## Architecture
- IPAM uses file-based locking via syscall.Flock on <dataDir>/.lock; all reads/writes to
  allocations.json must happen inside the locked section.
- netlinkwrapper.NetLink and nswrapper.NS are interfaces — inject them via network.New() for
  testability. pkg/ipam uses netlink directly (not through wrapper); this is intentional for now.
- The veth pair is created with hostVeth as LinkAttrs.Name and containerVeth as PeerName.
  The container-side peer exists at host level until LinkSetNsFd moves it into the netns.
  Therefore LinkByName(containerVeth) must be called BEFORE netns.Do, not inside it.

## Known gaps (as of project start)
- TeardownNetwork (DEL path) only deletes the host veth — it does NOT release the IPAM
  allocation. story ipam-release must be completed first so ReleaseAddr exists.
- No default route is installed in the container netns after IP assignment. Containers cannot
  reach the gateway until story network-default-route is done.
- IPTablesIface only exposes read-only methods (List, ListChains, ChainExists). Append/Delete
  are commented out. story iptables-masq must implement them before masquerade can be added.

## Testing notes
- Unit tests for pkg/ipam must set IPAMConfig.DataDir = t.TempDir() to avoid writing to
  /var/lib/cni/networks and to allow running without root.
- pkg/network tests need a mock NetLink that records LinkAdd/LinkDel calls; do NOT use the
  real netlink calls in unit tests (they require root and modify kernel state).
- Integration tests require root. Guard with: [ "$(id -u)" -eq 0 ] || { echo "skip: need root"; exit 0; }
- pkg/ipam tests require logging.InitStderr() before any code that calls logging.Logger; add
  a TestMain that calls logging.InitStderr() and os.Exit(m.Run()) to avoid nil-pointer panics.
- pkg/iptableswrapper TestInit requires root (iptables syscall); guard with
  if os.Getuid() != 0 { t.Skip(...) } to keep go test ./... green without root.

## iptables-masq
- IPTablesIface now has Exists/Append/AppendUnique/Delete; all delegating to go-iptables.
- Network.ipt is nil if NewIPTables fails (non-root); all iptables calls guarded by `if n.ipt != nil`.
- MASQUERADE rule: `-s <subnet> ! -o <bridge> -j MASQUERADE` in nat/POSTROUTING via AppendUnique.
- TeardownNetwork signature changed: added bridgeName string (second param); main.go passes conf.Bridge.
- Masquerade rule deleted in TeardownNetwork only when bridge has zero remaining MasterIndex ports.
- mockIPTables in iptables_test.go satisfies IPTablesIface (compile-time check via var _ IPTablesIface = (*mockIPTables)(nil)).

## unit-ipam
- Added `netlinkAdd func(link netlink.Link, addr *netlink.Addr) error` field to IPAM struct so
  BindNewAddr is testable without root; NewIPAM sets it to netlink.AddrAdd; tests inject noopAddrAdd.
- Test file extended (pkg/ipam/ipam_test.go) with table-driven TestBindNewAddr (allocates in range,
  two differ, reuse after release), TestReleaseStaleAllocations (invalid removed, valid kept),
  TestCheckStatus (error on no ranges, nil on valid range).
- writeAllocations helper seeds allocations.json directly for tests that don't need BindNewAddr.
- All 5 story acceptance cases pass without root (go test ./pkg/ipam/... -v).

## unit-network
- Added ipamIface interface (unexported) to network.go; Network.newIPAM field (func(*IPAMConfig) ipamIface) replaces all direct ipam.NewIPAM calls so IPAM is injectable in tests.
- New() default sets newIPAM to a closure: `i := ipam.NewIPAM(cfg); return &i`.
- mockNetLink stores links in a map; LinkAdd for *netlink.Veth also adds the PeerName link so both ends exist after creation.
- mockNetNS.Do() runs callback synchronously (no kernel namespace switch); Fd() returns 0.
- mockIPAM preset struct satisfies ipamIface; compile-time check via var _ ipamIface = (*mockIPAM)(nil).
- TestMain calls logging.InitStderr() to avoid nil Logger panic.
- All 4 acceptance tests pass without root: HappyPath, RollbackOnBridgeAttachFail, TeardownCallsLinkDel, CheckNetworkMissingHostVeth.

## network-default-route
- Default route is installed inside netns.Do after LinkSetUp using n.netlink.RouteAdd.
- Gateway parsed with net.ParseIP; net.ParseCIDR("0.0.0.0/0") or "::/0" used as Dst.
- If Ranges is empty or Gateway is empty string, RouteAdd is skipped (no error).
- gw.To4() == nil distinguishes IPv6 gateways to pick ::/0 as Dst.

## integration-script
- Script at scripts/integration-test.sh; invokes plugin binary directly (not cnitool) with CNI_COMMAND env vars.
- Plugin is installed to /opt/cni/bin before test; CNI conflist written to /etc/cni/net.d/99-eureka-test.conflist.
- ADD stdout is captured as JSON and validated with jq; CHECK config is built by jq-merging prevResult into the flat config.
- Bridge eureka0 is NOT deleted by TeardownNetwork — cleanup must run `ip link del eureka0` explicitly.
- Container ID must be ≥8 chars (first 8 used for host veth name prefix: veth<8 chars>).
- After a successful DEL, set CNI_CONFIG="" so the cleanup trap skips a redundant best-effort DEL.
